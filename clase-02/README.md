### Presentación de herramientas del curso: GitHub y un poco más. 

#### GitHub

GitHub es una **plataforma de gestión de repositorios remotos** que abstrae las complejidades técnicas de **Git** para facilitar la persistencia, trazabilidad y administración integral de cualquier base de código fuente. Al proporcionar una interfaz intuitiva y herramientas de visualización, este ecosistema trasciende la ingeniería informática, permitiendo que investigadores y profesionales de diversas disciplinas colaboren en proyectos mediante flujos de trabajo estructurados (*branching* y *pull requests*), garantizando la integridad de los datos y la consolidación de un portafolio técnico tanto en el ámbito académico como profesional.

Para materializar estas ventajas, GitHub ofrece entornos visuales que simplifican el flujo de trabajo. A continuación, se presentan los conceptos fundamentales para interactuar con este ecosistema, los cuales serán claves en este electivo troncal:

1. **El Repositorio (Tu Carpeta Inteligente):** Piensa en el "Repo" como la carpeta raíz de tu proyecto. Al crear uno, marca la opción **"Add a README file"**. Este será tu documento de presentación donde explicas de qué trata el proyecto en lenguaje sencillo.

2. **El Commit (La Foto del Momento):** A diferencia de un archivo de Word que guardas mil veces, en GitHub realizas un *Commit*. Es como tomar una fotografía de tus archivos en un punto exacto. **Regla de oro:** siempre escribe un comentario corto sobre qué cambiaste (ej: "Corregida la ortografía en la introducción").

3. **La Rama o "Branch" (Tu Borrador de Riesgo):** Si quieres probar una idea nueva sin alterar el trabajo principal, crea una *Branch*. Es una copia exacta donde puedes experimentar. Si el resultado es erróneo, se elimina; si es exitoso, se solicita su integración.

4. **El Pull Request (La Revisión por Pares):** Es el corazón de la colaboración. Cuando terminas una tarea en tu rama, generas un *Pull Request (PR)*. Es la solicitud formal para que el equipo revise tus cambios y, tras su aprobación, los fusione con el proyecto principal.

5. **Issues (Tu Lista de Tareas):** No todo es código. Usa la pestaña **Issues** para registrar errores, sugerencias o tareas pendientes. Funciona como un tablero de gestión vinculado directamente a la evolución del proyecto.

GitHub no es una herramienta exclusiva para programadores. En disciplinas como **las Ciencias Sociales o la Biología**, resulta invaluable para gestionar bases de datos (CSV), manuscritos en Markdown o protocolos de laboratorio, asegurando un historial de versiones inalterable que evita la pérdida accidental de información.

### GitHub Pages

GitHub ofrece la posibilidad de activar GitHub Pages por cada repositorio, permitiendo publicar contenidos de forma gratuita. Antes de activarlo, asegúrate de tener al menos un archivo llamado index.html o index.md en la raíz de tu repositorio; este será la "portada" de tu sitio web.

Sigue estos pasos para la activación:

1. **Ingresa a la Configuración**: Entra a tu repositorio y haz clic en la pestaña _Settings_ (Configuración), ubicada en la barra superior (icono del engranaje).

2. **Localiza la sección de Pages**: En el menú lateral izquierdo, bajo el apartado Code and automation, haz clic en _Pages_.
   
3. **Define la Fuente de Despliegue** (Build and deployment): En el apartado _Branch_, asegúrate de que esté seleccionada la opción _Deploy from a branch_.

- Selecciona la rama: Normalmente elegirás la rama `main` (la principal).

- Selecciona la carpeta: Elige `/(root)`.

- Haz clic en el botón _Save_ (Guardar).

4. **Espera el Procesamiento (GitHub Actions)**: GitHub comenzará a construir el servidor para tu sitio web. Cuando esté listo, verás un aviso resaltado en la parte superior de la pantalla de Pages indicando que el sitio está al aire (live).

5. **Vincula tu URL en la portada**: Vuelve a la pestaña principal del código (Code). En la barra lateral derecha busca la sección About y presiona el icono del engranaje vecino. En la ventana emergente, marca la opción "Use your GitHub Pages website". Al guardar, la dirección web de tu proyecto aparecerá visible para cualquier visitante del repositorio.
